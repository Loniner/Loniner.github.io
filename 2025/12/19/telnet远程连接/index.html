telnet远程连接

发表于2025-12-19|更新于2025-12-19|网络安全课程学习
|总字数:993|阅读时长:3分钟|浏览量:

system-view

进入系统视图，这是配置大部分功能的起点。

interface GigabitEthernet 0/0

进入千兆以太网接口 GigabitEthernet 0/0 的配置模式。这是设备上的一个物理接口，将作为Telnet管理的接入接口。

ip address 1.1.1.1 24

为该接口配置IP地址 1.1.1.1，子网掩码为 255.255.255.0（/24）。远程用户需要通过网络能访问到这个地址才能进行Telnet连接。

quit

退出接口配置视图，返回系统视图。

telnet server enable

全局启用Telnet服务器功能。 这是最关键的一步，没有这条命令，设备将拒绝所有Telnet连接请求。

local-user Loniner class manage

创建一个名为 Loniner 的管理类（class manage）本地用户。manage 类用户通常用于登录设备进行管理（如Telnet、SSH），区别于 network 类用户（用于PPPoE等网络服务）。

password simple 123456abcd

为该用户设置密码为 123456abcd。simple 表示密码以明文方式存储和显示，安全性较低。 在生产环境中强烈建议使用 password cipher 命令进行加密存储。

service-type telnet

指定该用户的服务类型为 telnet，即允许该用户使用Telnet方式登录设备。

authorization-attribute user-role level-15

为该用户授权用户角色为 level-15。这是网络设备的最高权限等级（等同于超级管理员），拥有此权限的用户可以执行所有配置和查看命令。

quit

退出本地用户配置视图，返回系统视图。

user-interface vty 0 4

进入虚拟终端线路（VTY）配置视图。vty 0 4 表示同时配置第0到第4条共5条VTY线路，即最多支持5个并发远程连接会话。

authentication-mode scheme

设置这5条VTY线路的认证模式为 scheme。

scheme（方案认证）：意味着登录时需要输入用户名和密码，设备会去检查本地创建的用户数据库（即刚才创建的 Loniner 用户）进行验证。

其他常见模式：

password： 只使用密码认证，无需用户名。

none： 无需认证（极其危险，切勿在生产环境使用）。

配置总结与连接流程
这段配置完成了一个标准的、基于用户名/密码认证的Telnet服务器设置。

远程连接流程如下：

管理员在电脑上打开Telnet客户端（如CMD，输入 telnet 1.1.1.1）。

连接请求到达设备的 GigabitEthernet 0/0 接口。

设备检查 telnet server enable 已开启，接受连接。

设备检查VTY线路 (vty 0 4) 的认证模式为 scheme，于是提示输入用户名和密码。

管理员输入：

用户名：Loniner

密码：123456abcd

设备在本地用户数据库中验证通过，并根据授权 (user-role level-15) 赋予用户最高管理权限。

登录成功，管理员可以开始命令行管理。

重要安全提醒与建议
密码明文问题： password simple 是隐患，应用 password cipher 加密。

协议安全性： Telnet协议本身不加密，所有通信（包括密码）都以明文传输，容易被网络窃听。在生产环境中，强烈建议使用更安全的SSH（Secure Shell）替代Telnet。

权限最小化： 不是所有管理员都需要 level-15 权限，应根据实际需要分配适当的权限等级。

访问控制： 可以通过配置ACL（访问控制列表）来限制只有特定的管理主机IP地址可以尝试Telnet连接，增加安全性。

接口可达性： 确保管理电脑的网络能够路由到 1.1.1.0/24 这个网段。

将Telnet升级为SSH的简要思路：

生成RSA或DSA密钥对。

将VTY线路的认证模式改为 scheme（通常已是）。

创建本地用户时，service-type 增加 ssh。

在VTY线路下指定支持的协议为 ssh。

全局启用 ssh server enable。
